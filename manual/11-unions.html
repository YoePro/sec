Unions

A union represents one active variant selected from a finite set. A variant may carry no data, one unnamed payload value, or a struct-like payload with named fields. 

type Message union {
    Quit
    Text(string)
    Move {
        x: int,
        y: int,
    }
}

Every value of Message contains exactly one active variant. The active variant and its payload are inspected with match.

Rule: A union is a closed, distinct named type. Its complete variant set is known at compile time.

Basic Syntax

A union is declared with type Name union:

type State union {
    idle
    running
    stopped
}

Variants are written inside the union body. Commas are optional after variants, and a trailing comma is allowed.

type State union {
    idle,
    running,
    stopped,
}

A union must have at least one variant.

type Empty union {
}
Invalid: Empty unions are not allowed.

Variant Kinds

Payload-less variants

A payload-less variant carries no additional data:

type State union {
    idle
    running
    stopped
}

Single-payload variants

A variant may carry exactly one unnamed payload value:

type Number union {
    Integer(int)
    Decimal(decimal)
}

Struct-like variants

A variant may carry a payload with named fields:

type Shape union {
    Circle {
        radius: decimal,
    }

    Rectangle {
        width: decimal,
        height: decimal,
    }
}
Rule: A variant may use either one unnamed payload or a struct-like named payload, but not both.

Construction

Payload-less construction

let state: State := State.idle

A payload-less variant does not accept arguments:

State.idle(1)
Invalid: State.idle does not accept a payload.

Single-payload construction

let number: Number := Number.Integer(42)

The constructor requires exactly one value of the declared payload type.

Number.Integer()
Number.Integer(1, 2)
Number.Integer("hello")

All three forms are invalid.

Struct-like construction

let rectangle: Shape := Shape.Rectangle {
    width: 10,
    height: 20,
}

All payload fields must be provided explicitly.

let rectangle: Shape := Shape.Rectangle {
    width: 10,
}
Invalid: The required field height is missing.

Unknown fields, duplicate fields and incompatible field values are rejected.

Construction result type

Every variant constructor produces the containing union type:

Expression Type
State.idleState
Number.Integer(42)Number
Shape.Circle { radius: 12 }Shape

A union variant does not create a separate source-level runtime type.

Variant Namespaces

Union variants belong to the union namespace.

type State union {
    idle
    running
}
State.idle
State.running

The names idle and running are not ordinary module symbols.

Variant names must be unique within one union:

type Invalid union {
    value
    value
}

Different unions may reuse the same variant name:

type State union {
    idle
    running
}

type Connection union {
    idle
    connected
}

Unions Are Distinct Types

Each union declaration creates a distinct named type.

type State union {
    idle
}

type Connection union {
    idle
}

State.idle and Connection.idle have different types.

let state: State := Connection.idle
Invalid: No implicit conversion exists between different union types.

Payload values also do not convert implicitly to or from the union:

let number: Number := 42
let value: int := Number.Integer(42)

Both declarations are invalid.

Explicit casts between unions and payload types are not supported.

Payload Rules

Every payload type must resolve to a valid type.

type Value union {
    Missing(UnknownType)
}

A struct-like payload field must have a unique name and a valid type.

type InvalidShape union {
    Rectangle {
        width: decimal,
        width: decimal,
    }
}

Duplicate payload types are allowed because the variants remain distinct:

type Value union {
    First(int)
    Second(int)
}

Value.First(1) and Value.Second(1) are different union values.

Multiple unnamed payload values are not supported initially. Use a struct-like payload instead:

type PairValue union {
    Pair {
        first: int,
        second: string,
    }
}

Assignment

A union value may be assigned only to the same union type.

let value: Number := Number.Integer(42)

Different generic instantiations are also distinct types:

let source: Option[string] := Option.Some("hello")
let target: Option[int] := source
Invalid: Option[string] cannot be assigned to Option[int].

Variables and Definite Assignment

A union has no implicit default variant.

let state: State

An immutable variable must be initialized when declared, so this is invalid.

A mutable union variable may be declared without an initializer:

let mut state: State
state = State.running

The variable must be definitely assigned before it is read.

Rule: The first declared variant is not selected automatically.

Equality

A payload-less union is comparable for equality.

if state == State.running {
    Start()
}

A union with payloads is comparable only when every possible payload type is comparable.

Variant identity is part of equality:

type Value union {
    First(int)
    Second(int)
}
Value.First(1) != Value.Second(1)

Different union types and different generic instantiations are never directly comparable.

Copy and Move Behavior

A union is copyable only when every possible payload is copyable.

  • A payload-less union is copyable.
  • A union containing only copyable payloads is copyable.
  • A union containing a non-copyable payload is not copyable.

Assignment follows the normal Sec copy and move rules. The compiler never silently clones a payload.

Matching Unions

match is the primary operation for inspecting a union value.

Payload-less variants

match state {
    idle => HandleIdle()
    running => HandleRunning()
    stopped => HandleStopped()
}

Single-payload variants

match number {
    Integer(value) => UseInt(value)
    Decimal(value) => UseDecimal(value)
}

Struct-like variants

area := match shape {
    Circle(circle) => circle.radius * circle.radius
    Rectangle(rectangle) => rectangle.width * rectangle.height
}

The complete struct-like payload is initially bound as one value. Field-level pattern destructuring is not supported yet.

// Not supported initially
Rectangle { width, height } => width * height

Exhaustiveness

A union is closed, so the compiler knows every possible variant.

type State union {
    idle
    running
    stopped
}
match state {
    idle => HandleIdle()
}
Invalid: The match is non-exhaustive because running and stopped are not handled.

A catch-all branch may cover all remaining variants:

match state {
    idle => HandleIdle()
    _ => HandleOther()
}

Duplicate variants, unknown variants and unreachable branches are rejected.

Payload Bindings

The type of a binding is derived from the matched variant.

type Number union {
    Integer(int)
    Decimal(decimal)
}
match number {
    Integer(value) => {
        // value has type int
        UseInt(value)
    }

    Decimal(value) => {
        // value has type decimal
        UseDecimal(value)
    }
}

Pattern bindings are immutable and exist only inside their branch.

A payload may be discarded explicitly:

match number {
    Integer(_) => HandleInteger()
    Decimal(value) => UseDecimal(value)
}

Match Guards

Union patterns may use normal where guards.

match number {
    Integer(value) where value > 0 => Positive(value)
    Integer(value) => OtherInteger(value)
    Decimal(value) => UseDecimal(value)
}

A guarded branch does not fully cover a variant. An unguarded branch or catch-all must handle the remaining values.

Generic Unions

Union declarations may use normal generic parameters.

type Option[T] union {
    Some(T)
    None
}

type Result[T, E] union {
    Ok(T)
    Err(E)
}

A concrete generic instantiation is a distinct union type.

let count: Option[int] := Option.Some(10)
let name: Option[string] := Option.Some("Sec")
let missing: Option[int] := Option.None

The shorter constructor form may be used when the concrete type can be inferred from the expected type or payload. 

let value: Option[int] := Option.Some(10)

The preferred fully explicit form is:

Option[int].Some(10)

This explicit constructor syntax may depend on compiler version support.

Nested Unions

A union may be declared inside an impl block.

type Token struct {
}

impl Token {
    type Kind union {
        identifier
        number
        symbol
    }
}

Outside the owner implementation, use the complete qualified name:

let kind: Token.Kind := Token.Kind.identifier

Inside impl Token, the owner name is omitted:

Kind.identifier

The complete nested names are:

Token.Kind
Token.Kind.identifier

Recursive Unions

Direct recursive payload storage is invalid because it would have infinite size.

type List[T] union {
    Node {
        value: T,
        next: List[T],
    }

    End
}
Invalid: List[T] directly contains another List[T].

Recursion through a finite-size indirection is valid:

type List[T] union {
    Node {
        value: T,
        next: Box[List[T]],
    }

    End
}

Other valid indirections may include references or unsafe pointers where their normal rules permit them.

Memory Layout

A concrete union value conceptually contains:

  • an active-variant tag,
  • storage large enough for the largest possible payload,
  • alignment sufficient for every possible payload.
struct UnionValue {
    tag
    payload_storage
}

Each variant receives an internal declaration-order index. These indices are compiler metadata.

Rule: Internal variant tags are not source-level integer values. Union values cannot be cast to or from integers.

Not Supported Initially

  • Anonymous unions
  • Open or extensible unions
  • Adding variants from an impl block
  • Multiple unnamed payload values
  • Field-level match destructuring
  • Variant-specific methods
  • Implicit payload-to-union conversion
  • Implicit union-to-payload conversion
  • Casts between unions and integers
  • Explicit numeric variant tags
  • Untagged or C-compatible unions
  • User-controlled union layout
  • Implicit default variants
  • Runtime reflection over variants

Common Errors

Constructing without the union namespace

let state: State := idle

Use:

let state: State := State.idle

Using a payload directly

let number: Number := 42

Use the variant constructor:

let number: Number := Number.Integer(42)

Reading payload data without matching

number.value

Use match to establish the active variant and bind its payload.

Incomplete match

match number {
    Integer(value) => UseInt(value)
}

Every possible variant must be handled.

Wrong construction style

Shape.Rectangle(10, 20)

Struct-like variants require named fields:

Shape.Rectangle {
    width: 10,
    height: 20,
}

Summary of Rules

  • A union represents exactly one active variant.
  • A union must be named and contain at least one variant.
  • Variants may be payload-less, single-payload or struct-like.
  • Variant names are unique within their union namespace.
  • Every constructor produces the containing union type.
  • No implicit conversion exists between a union and its payloads.
  • No implicit conversion exists between different union types.
  • Union payloads are inspected through match.
  • Union matches must be exhaustive.
  • Pattern bindings are immutable and branch-local.
  • A union has no implicit default variant.
  • Definite-assignment rules apply to uninitialized mutable union variables.
  • Generic union instantiations are distinct types.
  • Direct recursive storage is invalid.
  • Recursion through a defined finite-size indirection is allowed.
  • Copyability and comparability depend on every possible payload.
  • Internal variant tags are not exposed as integers.
