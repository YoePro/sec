Units

Units are semantic numeric types. They allow the compiler to distinguish values that use the same numeric representation but describe different quantities. 

unit m decimal physical
unit s decimal physical
unit SEK decimal other

A value measured in metres is not interchangeable with a value measured in seconds or Swedish kronor. Unit information remains part of the type and participates in assignment, arithmetic and conversion.

Rule: A unit describes meaning, while its numeric base type describes representation.

Unit Declaration

A unit is declared with a name, numeric base type and category:

unit m decimal physical
unit Item uint other

The general form is:

unit Name NumericType Category

The numeric base type must be numeric. Current forms include:

unit Count int other
unit Packet uint other
unit Temperature float physical
unit Distance decimal physical

The category is either physical or other.

Physical Units

Physical units participate in dimensional analysis. Their dimensions may represent length, time, mass, frequency or combinations of these quantities. 

unit m decimal physical
unit s decimal physical
unit kg decimal physical
unit Hz decimal physical

A physical unit may define its dimension, scale and system:

impl m {
    dimension: [length * 1]
    scale: 1
    system: SI
}

A derived physical unit may use several dimensions:

unit mps decimal physical

impl mps {
    dimension: [length * 1, time * -1]
    scale: 1
    system: SI
}

Domain Units

Units in the other category describe domain-specific quantities that are not part of a physical SI dimension. 

unit Item uint other
unit Packet uint other
unit Batch uint other

Each semantic quantity should use its own dimension:

impl Item {
    dimension: [item * 1]
    system: Domain
}

impl Packet {
    dimension: [packet * 1]
    system: Domain
}

Item and Packet may have the same storage type, but they remain incompatible units.

let items: Item := 10
let packets: Packet := 10
let total := items + packets
Invalid: Values with different semantic dimensions cannot be added.

Unit Identity

Every declared unit is a distinct semantic type.

unit m decimal physical
unit inch decimal physical

Both may describe length and use decimal storage, but they are not the same type. A value is not implicitly converted merely because the dimensions are compatible. 

let metric: m := 1
let imperial: inch := metric
Invalid: Unit conversions must be explicit.

Assignment

An untyped literal may initialize a unit when the literal fits its numeric base type.

let distance: m := 100
let duration: s := 10
let price: SEK := 5.90

Variables are not implicitly converted into another unit type.

let raw := 100
let distance: m := raw
Invalid: Values that already have a type require explicit conversion.

Normal assignment preserves exact unit identity:

let mut distance: m := 10
distance = m(20)

Explicit Conversion

Conversion into a unit uses normal explicit conversion syntax:

let raw := 100
let distance: m := m(raw)

Numeric casts and unit conversions are separate concepts. A numeric cast changes representation. A unit conversion changes semantic meaning while preserving dimensional correctness.

Compatible physical units may be converted when an exact scale relationship is known:

unit mm decimal physical

impl mm {
    dimension: [length * 1]
    scale: 0.001
    system: SI
}

Runtime or configured relationships require explicit conversion logic.

Addition and Subtraction

Addition and subtraction require compatible unit types.

let first: m := 10
let second: m := 20
let total := first + second

Different units cannot be added directly:

let distance: m := 10
let duration: s := 2
let invalid := distance + duration

Different named units normally remain distinct even when their dimensional vectors match.

Rule: Addition and subtraction preserve the original unit.

Multiplication and Division

Multiplication adds dimensional exponents. Division subtracts them.

m / s       // length * time^-1
m * m       // length^2
m / m       // dimensionless decimal

Equal dimensions cancel:

let first: m := 20
let second: m := 5
let ratio := first / second

ratio is dimensionless.

Multiplication or division by a plain scalar preserves the unit:

let price: SEK := 100
let doubled := price * 2
let half := price / 2

Some algebraic results may be rejected when they have no supported semantic meaning.

let invalid := price * price
Current rule: Squared currency values are rejected unless a valid use is defined explicitly.

Derived Units

When an expression produces a known dimension, the compiler may resolve the result to a declared unit with the same dimension and scale. 

unit m decimal physical
unit s decimal physical
unit mps decimal physical

impl mps {
    dimension: [length * 1, time * -1]
    scale: 1
    system: SI
}

let distance: m := 100
let duration: s := 10
let speed := distance / duration

speed may infer as mps.

When no declared unit matches, the result may remain an anonymous dimensioned numeric type such as decimal<m/s>.

Dimensions

A dimension is a vector of named bases and integer exponents.

[length * 1]
[time * 1]
[length * 1, time * -1]
[packet * 1, time * -1]

Dimensions determine algebraic compatibility.

ExpressionResulting dimension
m / s[length * 1, time * -1]
Hz[time * -1]
Packet / s[packet * 1, time * -1]
m / mdimensionless

Scale

Scale defines a unit relative to the canonical unit of its dimension.

unit mm decimal physical

impl mm {
    dimension: [length * 1]
    scale: 0.001
    system: SI
}
unit inch decimal physical

impl inch {
    dimension: [length * 1]
    scale: 0.0254
    system: Imperial
}

Because both units use the length dimension and have fixed scales, the compiler can validate an explicit conversion between them.

Unit Systems

The system name groups units but does not determine compatibility.

system: SI
system: Imperial
system: Currency
system: Domain
system: Other

Compatibility is determined by dimensions. Conversion is determined by scale and, for future offset units, by offset rules.

Currencies

Currencies are semantic domain units with separate dimensions.

unit SEK decimal other
unit EUR decimal other

impl SEK {
    dimension: [currency.SEK * 1]
    system: Currency
}

impl EUR {
    dimension: [currency.EUR * 1]
    system: Currency
}

SEK and EUR are not fixed-scale units. Their relationship depends on a runtime or configured exchange rate.

Rule: The compiler never invents a currency conversion rate.

Units on Register Fields

A multi-bit register field may use a unit annotation:

type MotorProtocol register[8] {
    Speed: bit[4]<rpm>
    Enabled: bit
    _: bit[3]
}

The field width defines the stored representation. The unit defines the semantic meaning of the decoded value. 

let speed: rpm := motorProtocol.Speed

An incompatible unit is rejected:

let distance: m := motorProtocol.Speed

Type Inference

Untyped integer literals infer as int by default:

let count := 10

Untyped decimal literals infer as decimal by default:

let value := 3.14

Floating-point storage must be selected explicitly:

let value: float64 := 3.14

A declared unit type provides context for an untyped literal:

let distance: m := 3.14

Common Errors

Implicit conversion from a variable

let raw := 10
let distance: m := raw

Adding incompatible units

let distance: m := 10
let duration: s := 2
let value := distance + duration

Assigning a unit to another semantic unit

let items: Item := 10
let packets: Packet := items

Assuming currencies convert automatically

let sek: SEK := 100
let eur: EUR := sek

Summary of Rules

  • A unit is a distinct semantic numeric type.
  • The numeric base type defines representation.
  • The unit category is physical or other.
  • Physical units may define dimensions, scale and system.
  • Domain units use independent semantic dimensions.
  • Untyped literals may initialize unit values in declaration context.
  • Variables require explicit conversion into another unit type.
  • Addition and subtraction require compatible units.
  • Multiplication and division perform dimensional algebra.
  • Equal dimensions cancel to a dimensionless numeric value.
  • Derived dimensions may resolve to declared unit types.
  • Unit systems do not determine dimensional compatibility.
  • Fixed exact scales may support explicit compile-time-verified conversion.
  • Runtime relationships such as currency rates remain explicit.
  • Units may be attached to multi-bit register fields.
  • No hidden unit conversion is performed.
